# dev-rules

プロジェクト横断で使う開発ルールのテンプレート集。

## 使い方

1. 新規プロジェクト開始時に RULES_TEMPLATE.md をコピー
2. プロジェクトの docs/RULES.md として配置
3. {{プロジェクト名}} 等のプレースホルダーをプロジェクト固有の値に置換
4. ファイルサイズ監視・技術的負債台帳にプロジェクトのファイルを記入

## ファイル構成

- RULES_TEMPLATE.md - 開発ルールテンプレート（本体）
- xamples/postcraft_rules.md - PostCraftでの適用例
- README.md - このファイル

## 核心ルール

このテンプレートの最重要ルールは **有識者議論プロセス** です。

- コードを書く前に、必ず有識者（専門家ペルソナ）を召喚して多角的に議論する
- スコアリングで8割以上になるまで実装に着手しない
- 「とりあえず書いてみる」を禁止し、設計品質を担保する
